# 二、技术架构文档 #

# 1. 技术栈 #

------

## 前端 ##

### React + TypeScript ###

原因：

- 生态成熟
- 动画支持强
- Tauri 兼容优秀

------

## UI ##

### TailwindCSS ###

------

## 动画 ##

### Framer Motion ###

------

# 2. 桌面框架 #

## Tauri 2.x ##

[Tauri 官方网站](https://tauri.app/?utm_source=chatgpt.com)

原因：

- 使用系统 WebView
- 内存占用远低于 Electron
- Rust 原生后端
- Windows API 集成方便 

------

# 3. 后端 #

## Rust ##

用于：

- Windows API 调用
- 状态管理
- IPC
- 后台服务

------

# 4. 系统架构 #

```
Frontend (React)
├─ Taskbar UI
├─ Dock UI
├─ Launcher UI
├─ Control Center UI
└─ Widgets

        ↓ IPC

Rust Backend
├─ Window Manager
├─ App State
├─ System API Layer
├─ Notification Watcher
└─ Config Manager

        ↓

Windows Native APIs
├─ Win32
├─ WinRT
├─ Core Audio
├─ WLAN API
└─ Bluetooth API
```

# 9. Tray 与通知策略 #

------

## 不接管 Tray ##

仅：

- 观察状态
- 同步状态

------

## 不重写通知系统 ##

完全保留：

```
Windows Notification Center
```

------

# 10. 多屏/DPI设计 #

必须支持：

- 125%
- 150%
- 200%
- 多显示器

------

# 11. 性能优化 #

------

## 避免 ##

- 高频轮询
- Explorer Hook
- DWM Hook
- 全局 DLL 注入

------

## 使用 ##

- Event-driven
- Win32 Callback
- 状态缓存

------

# 12. 安全性 #

禁止：

- 注入 Explorer
- Patch 系统 DLL
- Hook undocumented API

------

# 14. 开发建议 #

优先开发顺序：

1. Window Manager
2. Taskbar
3. Launcher
4. Control Center
5. Notification Badge
6. Dock
7. Workspace
