# Nebula Desktop 退出恢复验证记录

## 1. 测试范围

任务：`T6.4 退出恢复验证`

测试时间：2026-05-28

测试目标：

- 正常退出。
- 异常退出。
- Explorer 未被影响。
- Windows 原生任务栏和通知中心仍可用。

验收标准：

- 应用退出后系统桌面恢复正常。
- 不造成黑屏。
- 不造成 Explorer 崩溃。

## 2. 测试产物

```text
D:\data\Nebula_Desktop\src-tauri\target\release\nebula-desktop.exe
```

## 3. 测试方法

正常退出：

1. 启动 release 产物。
2. 等待主窗口创建。
3. 调用 `CloseMainWindow()`。
4. 等待进程退出。
5. 检查是否需要强制结束。

异常退出：

1. 启动 release 产物。
2. 等待主窗口创建。
3. 使用 `Stop-Process -Force` 结束 Nebula 进程。
4. 检查 Nebula 是否已退出。
5. 检查 Explorer 是否仍在运行。

Explorer 检查：

1. 测试前读取 `explorer.exe` 进程 ID。
2. 正常退出和异常退出后再次读取 `explorer.exe` 进程 ID。
3. 对比进程 ID 是否变化。

## 5. 未自动验证项

以下项目需要人工操作确认：

- Windows 原生任务栏点击是否完全正常。
- Windows Notification Center 是否可正常打开。
- 锁屏恢复后是否正常。

## 6. 当前结论

当前 release 产物满足基础退出恢复要求：

- 不替换 Explorer。
- 不做 Shell Replacement。
- 不 Hook Explorer。
- 不注入 Explorer。
- 正常退出不影响 Explorer。
- 强制结束 Nebula 不影响 Explorer。
