# Nebula Desktop 兼容性记录

## 1. 测试边界

本记录用于跟踪微信、QQ、企业微信、钉钉等常用软件在 Nebula Desktop 中的 Badge 行为。

必须遵守：

- 不 Hook Explorer。
- 不注入微信、QQ、企业微信、钉钉等进程。
- 不接管 Windows Notification Center。
- 不读取通知正文。
- 不读取聊天内容。
- 不使用 undocumented API。

当前 Badge 数据来源：

- `get_running_apps()`：窗口枚举、前台状态、最小化状态。
- `get_notification_indicators()`：窗口级 indicator，当前保守映射为 `normal`、`active`、`attention`。

当前不可声明支持的能力：

- 不保证能直接检测所有应用的真实未读消息数。
- 不保证能读取系统通知中心中的通知状态。
- 不保证能检测仅驻留托盘、未创建普通可见窗口的消息状态。
- 不读取任何消息内容或通知正文。

系统：Windows 11 Home 25H2

| 钉钉 | 否 | 未发现 | 当前环境无法验证 |

## 3. 检测矩阵

| 应用 | 可检测状态 | 不可检测或待验证状态 | 隐私风险 |
|---|---|---|---|
| 微信 | 主窗口存在、前台窗口、最小化窗口、进程路径 | 新消息是否一定触发窗口 Flash；托盘态未读；通知中心正文 | 当前方案不读取内容 |
| QQ | 主窗口存在、前台窗口、最小化窗口、进程路径 | 新消息是否一定触发窗口 Flash；托盘态未读；通知中心正文 | 当前方案不读取内容 |
| 企业微信 | 主窗口存在、前台窗口、最小化窗口、进程路径 | 多进程 Web 子窗口与主窗口的准确归并；新消息是否触发窗口状态变化 | 当前方案不读取内容 |
| 钉钉 | 待验证 | 当前未运行，无法判断窗口状态和新消息行为 | 当前方案不读取内容 |

## 4. Badge 行为记录

当前实现行为：

- `active`：窗口处于前台。
- `attention`：窗口处于最小化。
- `normal`：窗口存在但非前台、非最小化。

当前未完成的人工测试：

1. 微信收到新消息时，主窗口未前台且未最小化时是否出现可检测窗口状态变化。
2. QQ 收到新消息时，主窗口未前台且未最小化时是否出现可检测窗口状态变化。
3. 企业微信收到新消息时，主进程与 Web 子进程的窗口状态是否能稳定映射到同一个 Badge。
4. 钉钉运行后收到新消息时是否产生可枚举窗口状态变化。

## 5. 后续测试步骤

1. 启动 `src-tauri\target\debug\nebula-desktop.exe`。
2. 分别打开微信、QQ、企业微信、钉钉主窗口。
3. 将目标应用切到后台。
4. 使用另一账号向目标应用发送新消息。
5. 观察 Nebula Taskbar 上对应图标 Badge 是否从 `normal` 变为 `attention`。
6. 记录每个应用是否：
   - 有可见窗口。
   - 有稳定进程路径。
   - 新消息会触发窗口状态变化。
   - 新消息仅出现在系统通知中心或托盘中。
7. 不读取消息正文，不抓取通知内容。
