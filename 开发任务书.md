# Nebula Desktop 开发任务书

## 1. 任务目标

本任务书用于指导 Nebula Desktop 从文档阶段进入 MVP 开发阶段。

MVP 目标：

- 搭建 Tauri 2.x + React + TypeScript + TailwindCSS 工程。
- 实现 Windows 10 / Windows 11 桌面增强层基础能力。
- 完成 Taskbar、Launcher、Control Center、Notification Badge 初版。
- 保持 Overlay Architecture，不替代 Windows，不接管系统通知，不 Hook Explorer。

## 2. 版本范围

## 2.1 Phase 1: MVP

必须完成：

- 工程骨架。
- Rust 后端基础模块。
- React 前端基础 UI。
- Taskbar。
- Launcher。
- Control Center。
- Notification Badge 初版。
- 配置系统。
- 基础 SQLite 存储。
- Windows 10 22H2 兼容验证。
- Windows 11 25H2 本机验证。

不在 MVP 中完成：

- Workspace。
- Dock 高级动画。
- Widget。
- 插件系统。
- AI Command。
- 自定义通知系统。
- 全文文件搜索。

## 2.2 Phase 2

- Workspace。
- Dock。
- 多屏增强。
- Command Palette。
- 更完整的通知状态兼容。

## 2.3 Phase 3

- Widget。
- 插件系统。
- AI Command。
- Docker、Kubernetes、Git、SSH 等开发者工具集成。

## 3. 执行原则

开发过程中必须遵守：

- 不替换 `explorer.exe`。
- 不做 Shell Replacement。
- 不 Hook Explorer 内部实现。
- 不注入 Explorer 或其他进程。
- 不接管 Windows Notification Center。
- 不读取微信、QQ、企业微信、钉钉等聊天内容。
- 不读取通知正文。
- 不使用 undocumented API。
- 不做高频轮询。
- 不依赖 Windows 11 独占 API 实现核心功能。
- Windows 11 专属视觉能力必须提供 Windows 10 降级方案。

默认优先级：

1. 稳定性。
2. 兼容性。
3. 低占用。
4. 交互体验。
5. 视觉效果。

## 3.1 平台兼容要求

最低支持平台：

- Windows 10 22H2 x64。
- Windows 11 23H2 / 24H2 / 25H2 x64。

核心能力必须使用 Windows 10 可用方案：

- 窗口枚举、窗口标题、前台窗口、窗口激活使用 Win32。
- Taskbar 屏幕边缘预留使用 `SHAppBarMessage`。
- 音量控制使用 Core Audio API。
- WiFi 状态和入口使用 WLAN API 或 Windows 设置入口。
- 蓝牙能力优先运行时检测；不可用时跳转 Windows 设置。

视觉兼容要求：

- Windows 11 可以启用更现代的毛玻璃、圆角、阴影和材质增强。
- Windows 10 必须有稳定降级样式。
- 不允许为了视觉效果牺牲窗口稳定性。
- 不允许把 Mica、Acrylic、DWM 特性作为核心功能依赖。

## 4. 里程碑

| 里程碑 | 目标 | 产物 |
|---|---|---|
| M0 | 工程初始化 | 可启动的 Tauri + React 桌面应用 |
| M1 | 后端系统能力基础 | Window Manager、Config、IPC |
| M2 | Taskbar 初版 | 可显示固定应用和运行应用 |
| M3 | Launcher 初版 | 可搜索并启动应用 |
| M4 | Control Center 初版 | 可显示并调整音量，跳转系统设置 |

## 5. 任务拆分

## M0: 工程初始化

### T0.1 创建项目骨架

任务：

- 初始化 `package.json`。
- 初始化 Vite + React + TypeScript。
- 初始化 Tauri 2.x。
- 配置 TailwindCSS。
- 配置 Framer Motion。
- 配置基础目录结构。

产物：

```text
package.json
vite.config.ts
tsconfig.json
tailwind.config.js
src/
src-tauri/
```

验收标准：

- `npm run dev` 可启动前端。
- `cargo tauri dev` 或等效命令可启动桌面应用。
- 页面能显示 Nebula Desktop 基础 UI。

### T0.2 建立代码目录

任务：

- 创建前端目录：

```text
src/app/
src/ui/taskbar/
src/ui/launcher/
src/ui/control-center/
src/ui/shared/
src/ipc/
src/state/
src/styles/
src/utils/
```

- 创建后端目录：

```text
src-tauri/src/commands/
src-tauri/src/core/window/
src-tauri/src/core/appbar/
src-tauri/src/core/tray/
src-tauri/src/core/audio/
src-tauri/src/core/wifi/
src-tauri/src/core/config/
src-tauri/src/state/
src-tauri/src/utils/
```

验收标准：

- 目录结构与系统设计文档一致。
- 前后端模块边界清晰。

### T0.3 配置开发命令

任务：

- 增加 `dev`、`build`、`tauri:dev`、`tauri:build` 命令。
- 增加 TypeScript 检查命令。
- 增加 Rust 格式化和检查说明。

验收标准：

- 开发命令可执行。
- 构建失败时能定位到前端或后端问题。

## M1: 后端系统能力基础

### T1.1 IPC 基础设施

任务：

- 注册 Tauri Commands。
- 定义统一错误结构。
- 定义前后端共享数据模型。
- 前端封装 `invoke` 调用。

后端 Commands：

```text
get_app_version()
get_system_status()
get_config()
save_config(config)
```

验收标准：

- 前端能成功调用后端测试 Command。
- 错误返回结构一致。

### T1.2 Config Manager

任务：

- 实现默认配置。
- 实现配置文件读写。
- 实现配置合法性校验。
- 支持主题、Taskbar 位置、Dock 开关、Blur 开关。

验收标准：

- 首次启动生成默认配置。
- 修改配置后重启仍然生效。
- 非法配置能回退到默认值或给出错误。

### T1.3 Storage Layer

任务：

- 集成 SQLite。
- 创建 `pinned_apps`、`recent_items`、`app_cache` 表。
- 实现基础 CRUD。

验收标准：

- 固定应用能写入和读取。
- 最近项目能写入和读取。
- 数据库不存在时自动创建。

### T1.4 Window Manager 初版

任务：

- 使用 Win32 枚举顶层窗口。
- 获取窗口标题。
- 获取进程 ID。
- 获取前台窗口。
- 获取窗口最小化状态。
- 过滤不可见窗口和无标题噪声窗口。

后端 Commands：

```text
get_running_apps()
get_foreground_window()
```

验收标准：

- 能列出 VSCode、浏览器、资源管理器等常用窗口。
- 当前激活窗口能正确标记。
- 不显示明显系统噪声窗口。

## M2: Taskbar 初版

### T2.1 Taskbar 布局

任务：

- 实现 Taskbar 主组件。
- 支持 Top 和 Bottom。
- 预留 Left 和 Right 的布局接口。
- 实现毛玻璃或半透明降级、圆角、Hover 动画。

验收标准：

- Taskbar 在主屏顶部或底部显示。
- UI 不遮挡核心系统区域。
- 切换位置后布局正确。
- Windows 10 上无 Windows 11 材质时仍然可正常显示。

### T2.2 固定应用

任务：

- 实现固定应用列表 UI。
- 从 SQLite 或默认配置读取固定应用。
- 点击固定应用启动程序。
- 支持图标显示。

验收标准：

- 至少支持浏览器、Terminal、文件管理器、IDE 四类固定应用。
- 应用不存在时显示明确状态。

### T2.3 运行应用显示

任务：

- 调用 `get_running_apps()`。
- 显示运行中应用。
- 标记激活窗口。
- 标记最小化状态。

验收标准：

- 打开和关闭窗口后 Taskbar 状态能更新。
- 激活窗口标记准确。
- 最小化窗口有独立状态。

### T2.4 窗口激活控制

任务：

- 实现点击 Taskbar 图标激活窗口。
- 已最小化窗口点击后还原。
- 多窗口同应用先显示基础合并策略。

后端 Commands：

```text
activate_window(window_id)
restore_window(window_id)
minimize_window(window_id)
```

验收标准：

- 点击运行应用图标能切换窗口。
- 最小化窗口能还原。

## M3: Launcher 初版

### T3.1 Launcher UI

任务：

- 实现 Spotlight/Raycast 风格输入框。
- 实现结果列表。
- 实现键盘上下选择。
- 实现 Enter 执行。
- 实现 Esc 关闭。

验收标准：

- 快捷键或 Taskbar 按钮能打开 Launcher。
- 输入时结果列表稳定显示。
- 键盘操作流畅。

### T3.2 应用搜索

任务：

- 扫描开始菜单快捷方式。
- 建立内存索引。
- 支持按名称搜索。
- 支持启动搜索结果。

后端 Commands：

```text
search_apps(query)
launch_app(app_id)
```

验收标准：

- 能搜索并打开常用应用。
- 搜索响应不明显卡顿。
- 不扫描全盘。

### T3.3 最近项目

任务：

- 读取本地最近项目记录。
- Launcher 中显示最近项目。
- 打开项目后更新时间。

验收标准：

- 最近项目可显示。
- 点击最近项目可打开。

## M4: Control Center 初版

### T4.1 Control Center UI

任务：

- 实现控制中心面板。
- 实现音量、WiFi、蓝牙、亮度、夜间模式、电源模式分组。
- 实现深色模式视觉。
- 实现展开和关闭动画。

验收标准：

- Taskbar 按钮可打开 Control Center。
- 面板不会与 Taskbar 发生错位。
- 动效流畅。

### T4.2 音量控制

任务：

- 使用 Core Audio API 读取系统音量。
- 使用 Core Audio API 设置系统音量。
- 前端 Slider 与后端状态同步。

后端 Commands：

```text
get_volume()
set_volume(value)
```

验收标准：

- 能正确显示当前系统音量。
- 拖动 Slider 能调整系统音量。

### T4.3 系统设置入口

任务：

- 实现 WiFi 设置入口。
- 实现蓝牙设置入口。
- 实现显示设置入口。
- 实现电源设置入口。

验收标准：

- 点击入口能打开 Windows 原生设置页。
- 不强行重写系统设置。

## M5: Notification Badge 初版

### T5.1 Badge UI

任务：

- 在 Taskbar 图标上显示通知点。
- 支持普通、活跃、注意三种状态。
- 保持视觉克制，不显示内容。

验收标准：

- Badge 不影响图标点击。
- Badge 与不同尺寸图标对齐正常。

### T5.2 注意状态检测

任务：

- 封装窗口 Flash / Attention 状态检测。
- 将状态映射到应用。
- 向前端推送 Badge 状态。

后端 Commands / Events：

```text
get_notification_indicators()
notification_indicator_changed
```

验收标准：

- 应用请求注意时能显示 Badge。
- 不读取通知正文。
- 不读取聊天内容。

### T5.3 微信/QQ兼容测试

任务：

- 测试微信新消息时 Badge 行为。
- 测试 QQ 新消息时 Badge 行为。
- 测试企业微信新消息时 Badge 行为。
- 记录不可检测场景。

验收标准：

- 至少明确每个应用可检测和不可检测的状态。
- 不引入 Hook 或隐私读取。

## M6: 稳定性与性能验证

### T6.1 性能验证

任务：

- 记录启动时间。
- 记录空闲 CPU。
- 记录内存占用。
- 检查窗口枚举频率。

验收标准：

- 启动时间 < 2s。
- 空闲 CPU < 1%。
- 内存 < 200MB。

### T6.2 Windows 版本兼容验证

任务：

- 在 Windows 10 22H2 上验证启动、退出、Taskbar、Launcher、Control Center。
- 在 Windows 11 25H2 上验证启动、退出、Taskbar、Launcher、Control Center。
- 对比两套系统的窗口透明、阴影、圆角、焦点行为。
- 记录 Windows 10 降级样式是否正常。

验收标准：

- Windows 10 和 Windows 11 都能启动应用。
- 核心功能在两个系统上均可用。
- Windows 10 上视觉降级不影响操作。
- Windows 11 上增强视觉不影响稳定性。

### T6.3 多屏 DPI 验证

任务：

- 测试 125%、150%、200% 缩放。
- 测试主屏和副屏。
- 测试显示器插拔。
- 测试 Taskbar Top / Bottom。

验收标准：

- UI 不错位。
- AppBar 工作区计算正确。
- 显示器变化后可恢复布局。

### T6.4 退出恢复验证

任务：

- 测试正常退出。
- 测试异常退出。
- 测试 Explorer 未被影响。
- 测试 Windows 原生任务栏和通知中心仍可用。

验收标准：

- 应用退出后系统桌面恢复正常。
- 不造成黑屏。
- 不造成 Explorer 崩溃。
